## Concepts

- **Input requirement**: arriba requires SAM/BAM files generated by the STAR aligner (version 2.7.0 or later) with specific flags enabled: `--chimSegmentMin 2` and `--outSAMstrandField intronMotif`. Using alignments from other aligners will cause runtime failures or silent false negatives.
- **Reference annotations**: arriba relies on a genome FASTA, gene annotation GTF, and a blacklists file. The blacklist (`blacklist.tsv`) filters out artifacts and recurrent false positives; without it, fusion output will contain many spurious calls that would otherwise require manual curation.
- **Output format**: Results are written as a tab-separated file with columns including `#FusionName`, `LeftGene`, `RightGene`, `JunctionReadCount`, `SpanningReadCount`, `ReadingFrame`, and `Direction`. The confidence rating (high/medium/low/discarded) is derived from readcounts and structural plausibility, not from arbitrary thresholds.

## Pitfalls

- **Omitting STAR chimeric flags**: Running STAR without `--chimSegmentMin 2` produces no chimeric alignments, meaning arriba will report zero fusions despite valid data. This is a silent failure—you must verify the STAR log for `CHimeric` segment counts or the output will be empty.
- **Using mismatched genome indices**: If the STAR index was built with a different reference version than the one supplied to arriba, the coordinates will be inconsistent, producing fusions with incorrect chromosomes or positions that appear in the output but cannot be validated.
- **Neglecting strand information**: When `intronMotif` is not set in the STAR command, the strand of fusion junctions becomes ambiguous, causing arriba to assign incorrect reading frames and mis-orient fusions, which propagates into downstream interpretation errors.
- **Running without blacklist filtering**: Failing to provide the blacklist file results in hundreds of artefactual fusion calls from homologous genomic regions, dramatically increasing manual review time and reducing the precision of downstream analysis.

## Examples

### Detect gene fusions from a STAR-aligned BAM file

**Args:** -x reads.bam -a annotations.gtf -g genome.fa -b blacklist.tsv -o fusion_output.tsv

**Explanation:** This runs fusion detection on the input BAM using the annotation GTF, genome FASTA, and blacklist for filtering; results are written to the specified output file.

### Run fusion detection with relaxed confidence thresholds

**Args:** -x reads.bam -a annotations.gtf -g genome.fa -b blacklist.tsv -O fusion_output.tsv -m 2

**Explanation:** The `-m 2` flag lowers the minimum junction read count threshold, allowing detection of lower-expressed fusions at increased risk of false positives.

### Generate fusion visualization graphics

**Args:** -i fusion_output.tsv -o fusion_graphs/ -g genome.fa

**Explanation:** The companion binary `arriba-draw` takes arriba's output and generates PNG images showing fusion structure, supporting reads, and genomic context for each fusion.

### Use a custom blacklist for specific tissue types

**Args:** -x reads.bam -a annotations.gtf -g genome.fa -b custom_blacklist.tsv -o fusion_output.tsv

**Explanation:** Supplying a tissue-specific blacklist (curated from prior samples) reduces artefactual calls that are recurrent in that sample type, improving precision.

### Output only high-confidence fusions

**Args:** -x reads.bam -a annotations.gtf -g genome.fa -b blacklist.tsv -o fusion_output.tsv -F high

**Explanation:** The `-F high` flag filters output to retain only fusions rated high confidence, discarding medium and low-confidence calls for downstream applications requiring maximum specificity.

### Extract fusions supporting a specific gene

**Args:** -x reads.bam -a annotations.gtf -g genome.fa -b blacklist.tsv -o fusion_output.tsv | grep -w "MYC"

**Explanation:** Piping the output through grep to isolate fusions involving a gene of interest (e.g., MYC) for rapid targeted analysis without manually scanning the full file.
