## Concepts

- **Input Formats**: Accepts both SAM (text) and BAM (binary) alignment files. The tool automatically detects the format based on the file extension (.sam or .bam) or magic number in the file header.
- **Sort Order Types**: Validates three primary sort orders — `coordinate` (default, sorted by reference name and position), `queryname` (sorted by read name), and `unsorted`. Some tools require specific sort orders; using the wrong one causes failures.
- **Validation Behavior**: When checking a file, the tool reads the header's `@HD` tag for the `SO` field first. If missing or invalid, it performs a full scan of alignment records to determine actual sort order and reports discrepancies.
- **Output Modes**: Returns exit code 0 if sort order matches the expected type, non-zero exit codes (1 for mismatch, 2 for error) for reporting in pipelines. Optional verbose output describes the actual and expected sort orders.

## Pitfalls

- **Assuming Header is Truth**: Always verify the sort order manually if the header's `@HD SO:` tag is missing or contradictory. Some files generated by older tools contain incorrect header information, causing downstream tools to fail silently or produce wrong results.
- **Mismatching Sort Order for Downstream Tools**: Using a queryname-sorted file with tools expecting coordinate-sorted input (e.g., `samtools mpileup`) will cause errors or produce incorrect variant calls. Always match the required sort order to your analysis pipeline.
- **Input File Corruption**: Running `check-sort-order` on a truncated or corrupted BAM file produces misleading errors. Validate file integrity separately using `samtools flagstat` before relying on sort order checks in critical pipelines.

## Examples

### Check if a BAM file is coordinate sorted
**Args:** `-s coordinate input.bam`
**Explanation:** Validates that the BAM file is sorted by reference coordinate, which is required for many variant calling pipelines.

### Verify sort order of a SAM file with verbose output
**Args:** `-v input.sam`
**Explanation:** Prints detailed information about the detected sort order and header details, useful for debugging.

### Check if a file is queryname sorted
**Args:** `-s queryname alignment.bam`
**Explanation:** Confirms the BAM file is sorted by read name, which is required for tools like `samtools fixmate`.

### Force validation without trusting header
**Args:** `-f -s coordinate input.bam`
**Explanation:** Performs a full record scan to determine actual sort order, ignoring potentially corrupted header information.

### Validate and print exit code only
**Args:** `-q -s coordinate input.bam`
**Explanation:** Outputs nothing if sort order matches; useful in shell scripts where only the exit code matters for conditional logic.

### Check multiple files in batch
**Args:** `-s coordinate file1.bam file2.bam file3.bam`
**Explanation:** Reports the status of each file sequentially, allowing quick validation of multiple alignment files in one run.
