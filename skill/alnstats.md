## Concepts

- **Input Format**: alnstats reads MAQ's native `.li` alignment index files (binary format storing read name, chr, position, mapping quality, and pairing info) or SAM/BAM files when using the `maq` codec flag. Multiple input files can be supplied to aggregate statistics across datasets.
- **Quality Filtering**: The `-q` flag filters alignments by minimum mapping quality threshold (0-255), excluding reads below the specified value from all statistics—this is critical for removing unreliable alignments before computing coverage or error profiles.
- **Output Modes**: alnstats operates in summary mode (`-s`) showing aggregate statistics (total reads, mapped fraction, mean quality) or detail mode (`-d`) outputting per-read metrics, allowing downstream parsing for custom analyses.
- **Paired-End Handling**: When alignment files contain paired-end data, flags `-1` and `-2` restrict analysis to first or second reads in pairs respectively, enabling sex-specific or library-prep bias detection in downstream scripts.

## Pitfalls

- **Unfiltered Low-Quality Reads**: Running alnstats without `-q` includes zero-quality alignments (unmapped or ambiguous reads), artificially lowering mean mapping quality and inflating the "mapped" read count—always apply `-q` ≥ 10 for meaningful statistics in variant calling workflows.
- **Misinterpreting Detail Output**: Using `-d` generates per-read lines that require parsing; forgetting to redirect to a file (`-o output.txt`) prints to stdout and may truncate large datasets due to terminal buffer limits.
- **Incompatible File Versions**: MAQ's `.li` format changed between versions; feeding files generated by `maq map` (old version) into newer alnstats binaries causes silent failures or garbled statistics—validate files with `maq verify` first.
- **Ignoring Read Pair Orientation**: For paired-end libraries, alnstats does not automatically detect FR/FF orientation by default; running without inspecting pairing flags can mislead coverage calculations in duplicate-sensitive applications like copy number variation.
