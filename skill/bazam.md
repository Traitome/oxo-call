## Concepts

- Bazam reads SAM-formatted alignments and performs indel realignment using the reference genome, producing a sorted and indexed BAM file as output. The realignment algorithm mimics the GATK indel realigner to reduce false positive variant calls in downstream analysis.
- Bazam requires a reference genome in FASTA format with a `.amb`, `.ann`, `.bwt`, `.pac`, and `.sa` index (generated by BWA index or the `bazam-index` companion tool). Without a valid index, bazam throws an exception and halts.
- Memory usage in bazam is proportional to the region window size and read depth; larger `--batchSize` values increase memory consumption but reduce I/O overhead. The tool processes reads in streaming batches rather than loading the entire SAM file into memory.
- The companion binary `bazam-rewrap` converts the intermediate realigned BAM back to SAM format when SAM output is desired. The two tools are designed to be piped together.
- Bazam is primarily used with 454/Sequel and similar NGS platform data where indel realignment is most beneficial, but it accepts any SAM file produced by any aligner provided proper sorting and read group metadata are present.

## Pitfalls

- Running bazam without a pre-indexed reference genome causes a crash with a cryptic "Reference sequence not indexed" error. Always run `bazam-index ref.fa` on the FASTA file before the first realignment job.
- Specifying an incorrect `--insertSize` for paired-end libraries causes misaligned realignment decisions, resulting in incorrectly anchored alignments and degraded downstream variant calling accuracy.
- Bazam does not validate read group (`@RG`) tags in the SAM header; reads without a read group are processed but may cause compatibility issues with GATK tools downstream, leading to silent failures or missing read group metadata in the output.
- Setting `--batchSize` too large for available system RAM causes an OutOfMemoryError, aborting the job with no partial output written. Monitor memory usage and use conservative batch sizes on shared systems.
- Bazam requires input SAM files to be coordinate-sorted; providing a query-sorted or unsorted SAM file causes alignment anchoring failures and produces malformed BAM output.

## Examples

### Realign a SAM file against a reference genome

**Args:** `-R ref.fa input.sam -o realigned.bam`
**Explanation:** This command reads the SAM file, realigns reads using the reference genome in ref.fa, and writes the sorted, realigned BAM output to realigned.bam.

### Realign with a custom batch window size to control memory usage

**Args:** `-R ref.fa input.sam --batchSize 5000 -o realigned.bam`
**Explanation:** Setting a batch size of 5000 limits the number of reads processed per window, controlling memory consumption on systems with limited RAM.

### Realign paired-end data with insert size specification

**Args:** `-R ref.fa input.sam --insertSize 250 -o realigned.bam`
**Explanation:** Specifying an insert size of 250 informs the realignment engine of the expected library fragment length, improving paired-end anchoring accuracy during realignment.

### Realign and convert output back to SAM format

**Args:** `-R ref.fa input.sam -o - | bazam-rewrap -o realigned.sam`
**Explanation:** Streaming the BAM output through bazam-rewrap converts the realigned BAM back to SAM format, useful when downstream tools expect SAM input.

### Realign with multithreaded processing for large files

**Args:** `-R ref.fa input.sam --numThreads 8 -o realigned.bam`
**Explanation:** Allocating 8 threads enables parallel realignment across regions, significantly reducing wall-clock time for large SAM files on multi-core systems.

### Validate reference index before running realignment

**Args:** `bazam-index ref.fa`
**Explanation:** Running the companion index command on the reference FASTA builds the required BWA-style suffix array index, preventing runtime crashes due to missing reference indices.
