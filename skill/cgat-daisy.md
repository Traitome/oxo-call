## Concepts

- Allele-specific expression (ASE) detection relies on heterozygous SNPs as natural markers: each het-SNP serves as an internal control where the reference and alternate alleles should be expressed in equal proportion in the absence of regulatory effects. `cgat-daisy` counts mapped reads spanning every het-SNP and tests whether the observed ratio deviates significantly from 0.5.
- Input format is **sample-sheet based**: a plain-text sample sheet (columns: `track`, `condition`, `bam`, `replicate_id`) tells `cgat-daisy` which BAM files to process and how to group them. Condition labels in this sheet drive the statistical comparison, so mismatched or inconsistent labels silently collapse groups or produce zero-count tests.
- Output includes per-variant **Fisher-exact odds ratios**, **combined p-values** across replicates, **allelic fractions**, and **adjusted q-values** (Benjamini–Hochberg). Only variants with a user-specified minimum read depth are retained; variants below the depth threshold are removed before multiple-testing correction to avoid inflation of the null distribution.
- The tool supports **paired analysis**: when a `paired_column` is specified, `cgat-daisy` pairs samples row-by-row (e.g., tumour–normal from the same individual) before computing differences, dramatically improving power when inter-individual variance is high.
- Results can be exported in **CSV**, **compressed TSV**, or **bigWig** format for genome-browser visualisations. The bigWig export applies a smoothing window by default, which may obscure sharp allele-switch events at splice junctions; disable smoothing with `--bigwig-no-smooth` when inspecting individual SNP resolution.

## Pitfalls

- **Inconsistent sample-sheet condition labels** cause silent group collapse: if "treated" appears as "treat" in one row, `cgat-daisy` treats them as separate conditions and either runs an unintended comparison or exits with a "no replicates" error. Always audit condition labels with `cut -f2 samplesheet.tsv | sort -u` before running.
- **Low read depth at het-SNPs** produces noisy allelic ratios; applying only a fold-change filter (e.g. `--fc 2.0`) on sites with 3–5 reads can falsely nominate them as ASE. Always set a minimum `--depth` threshold (≥20 total reads recommended) AND a fold-change filter jointly.
- **Specifying a homozygous site list** means no reference/alternate allele distinction exists, so the tool will report a constant allelic fraction of 0.0 or 1.0 across all samples, yielding no significant results. Supply only genuinely heterozygous SNPs (HWE p > 0.01 in the population) as the input variant list.
- **Missing required index files**: `cgat-daisy` reads BAM files via `pysam` and requires a corresponding `.bai` (BAI) index alongside every `.bam`. If the index is absent or stale (older timestamp than the BAM), the tool silently returns zero counts for the unindexed samples without raising an error.
- **Legacy VCF version mismatch**: variant lists generated by older CGAT pipelines may use VCF v4.0 syntax with unquoted INFO fields. The current `cgat-daisy` parser expects VCF v4.1+ and will silently skip malformed records; use `--input-format vcf4.1` explicitly to enforce stricter validation and catch missing records early.

## Examples

### Detect allele-specific expression between treated and control RNA-seq samples
**Args:** `ase --bam treated.bam --bam-control control.bam --variants het_snps.vcf --output ase_results.csv`
**Explanation:** Counts reference and alternate allele reads at each het-SNP in both BAM files, performs a Fisher exact test, and exports the per-SNP odds ratios and p-values to CSV for immediate review.

### Run an allelic imbalance screen with a high depth threshold to exclude noisy sites
**Args:** `screen --bam tumour_rep1.bam --bam-normal normal_rep1.bam --variants het_snps.vcf --depth 30 --output ai_screen.csv`
**Explanation:** Applies a 30-read minimum-depth filter so that sites supported by fewer than 30 total reads are dropped, reducing false-positive ASE calls arising from low-coverage stochasticity.

### Compare two conditions listed in a sample sheet with output in compressed TSV
**Args:** `ase --sample-sheet conditions.tsv --variants het_snps.vcf --output-compress tsv.gz --output ase_compressed.tsv.gz`
**Explanation:** Reads the sample sheet to group BAMs by condition label, then writes a gzip-compressed TSV containing odds ratios, allelic fractions, and q-values for all variants.

### Perform paired analysis using individual_id column for tumour–normal matching
**Args:** `paired --bam tumour.bam --bam-control normal.bam --variants het_snps.vcf --paired-column individual_id --output paired_ase.csv`
**Explanation:** Pairs tumour and normal samples that share the same `individual_id` before computing differences, eliminating inter-individual background variation and increasing statistical power.

### Annotate significant ASE results with a reference gene track for downstream GO over-representation
**Args:** `annotate --variants significant_ase.vcf --genes-gtf annotation.gtf --output annotated_ase.csv`
**Explanation:** Overlays significant het-SNPs onto gene annotations from the GTF and outputs chromosome, position, gene name, and biotype for each variant, enabling downstream Gene Ontology or pathway enrichment analysis.
