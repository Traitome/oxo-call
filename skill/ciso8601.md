## Concepts

- **ISO 8601 Support**: ciso8601 parses standard ISO 8601 formats including basic (`20231225T143000`) and extended (`2023-12-25T14:30:00`) notations, with support for fractional seconds (e.g., `.123456`), timezone offsets (`+0100` or `Z`), and week-based dates (`2023-W52`).
- **Output Structures**: The C library converts parsed datetimes into `struct tm` (calendar time), `struct timespec` (with nanoseconds), or `time_t` (Unix timestamp), while the Python module returns `datetime.datetime` objects with timezone awareness when timezone information is present.
- **Error Handling**: The parser returns integer error codes (0 for success, negative values for specific errors) and uses output parameters to return parsed values, requiring pre-allocated output structures passed by reference.
- **Thread Safety**: The library is reentrant and thread-safe, allowing concurrent parsing calls without mutex protection, though each call requires separate output structures.

## Pitfalls

- **Ignoring Error Codes**: Failing to check the return value from parse functions can silently propagate invalid data into your program. Always verify the return code is zero before using the parsed result.
- **Timezone Ambiguity**: Parsing datetimes without timezone information (e.g., `2023-12-25T14:30:00`) defaults to UTC in Python bindings but may retain timezone-naive status in C structures, leading to inconsistent behavior across modules.
- **Fractional Truncation**: The C library's `struct timespec` output truncates microseconds to nanosecond precision, while sub-microsecond fractional components are lost, potentially causing precision loss in high-resolution timestamp applications.
- **Buffer Overflows**: When using the `ciso8601_to_time_t` or similar functions with fixed-size output buffers, providing insufficient buffer space leads to undefined behavior or memory corruption.
