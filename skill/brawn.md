## Concepts

- **Index dependency**: `brawn` cannot operate without a pre-built index generated by `brawn-build`. The index files (.bwt, .sa, .pac, .ann, .rbwt, .rsa, .rpac, .rhs) must exist in the same directory as the reference or be explicitly specified.
- **Input format flexibility**: Accepts FASTQ (single-end), FASTQ paired-end (two files), and FASTA formats. Supports gzipped (.gz) and bzipped (.bz2) compressed inputs via stream decompression.
- **Output modes**: Three alignment modes—mem (original maximal exact match), samse (single-end from MEMs), and sampe (paired-end from MEMs)—control the algorithm and pairing behavior. Output is SAM format by default; piping to `samtools view` converts to BAM.
- **Scoring parameters**: Mismatch penalty (`-A`), match reward (`-a`), gap opening (`-o`), and gap extension (`-e`) control alignment sensitivity. Lower gap penalties increase sensitivity but reduce speed; higher penalties improve runtime at the cost of accuracy.
- **Thread parallelism**: Multi-threading via `-t` divides work across CPU cores, scaling nearly linearly for large datasets. Memory usage per thread scales with index size and read depth.

## Pitfalls

- **Running alignment without index**: Attempting `brawn mem ref.fa reads.fq` without prior `brawn-build ref.fa ref` produces an error or silently produces garbage alignments, leading to invalid downstream variant calls.
- **Mismatched reference versions**: Using an index built on an older reference genome build while aligning new reads causes systematic misalignment of updated loci, producing false-positive or false-negative variants in chrM and segmental duplication regions.
- **Excessive threads on memory-constrained systems**: Setting `-t 32` on a 64GB RAM machine with a 3GB human genome index causes OOM killer events; each thread consumes ~300MB additional buffer. For a 3GB index, stick to `-t 8` or fewer.
- **Forgetting paired-end read ordering**: Passing interleaved paired-end files to `brawn mem` without `-p` interprets read pairs as independent single-end reads, halving effective coverage and producing spuriousmate discordances.
- **Ignoring RG tags in BAM**: Omitting `@RG` headers in input or `-r` read group specification makes downstream GATK pipelines unable to deduplicate or adjust for library quality, biasing variant quality (QD) scores downward.

## Examples

### Build a reference genome index
**Args:** `ref.fa brawn-ref`
**Explanation:** Creates the binary index files required for subsequent alignment. The reference must be in FASTA format; index naming convention enables shorthand reference specification.

### Align single-end FASTQ reads using MEM algorithm
**Args:** `-t 8 brawn-ref reads.fq`
**Explanation:** Uses the maximal exact match algorithm, the recommended mode for Illumina 100bp+ reads. Multi-threading with 8 cores accelerates throughput on multi-core workstations.

### Align paired-end FASTQ files with output to SAM
**Args:** `-t 16 -o 11 -e 2 brawn-ref left.fq right.fq > aligned.sam`
**Explanation:** Sets higher gap-opening penalty (11) and lower gap-extension penalty (2), typical for RNA-seq or reads with confident base qualities. Output redirection writes SAM directly.

### Align paired-end reads from an interleaved FASTQ file
**Args:** `-t 8 -p brawn-ref interleaved.fq`
**Explanation:** The `-p` flag instructs `brawn` to treat the input as paired-end data in a single file, where read1/read2 pairs are ordered consecutively. Required for split-lane concatenation.

### Align with custom mismatch and gap penalties
**Args:** `-t 4 -A 2 -a 1 -b 4 -q 2 brawn-ref reads.fq`
**Explanation:** Reduces mismatch penalty to 2 and match reward to 1 relative to defaults, making the aligner more permissive for highly divergent sequences like viral reads or degraded ancient DNA.

### Convert SAM output to sorted BAM in one pipeline
**Args:** `- bwamem -t 12 ref.fa reads.fq | samtools view -bS - | samtools sort -o sorted.bam -`
**Explanation:** Chains `brawn` stdout directly to `samtools` for in-memory BAM conversion and coordinate sorting, avoiding intermediate SAM file creation and saving disk I/O.

### Align with read group tagging for downstream GATK
**Args:** `-t 8 -R '@RG\tID:lane1\tSM:sample1\tPL:ILLUMINA' brawn-ref reads.fq`
**Explanation:** Embeds read group metadata in output SAM header, enabling GATKBaseRecalibrator to correct quality scores per-lane and sample, critical for variant quality (VQSR) accuracy.
