## Concepts

- **Read Count Normalization**: bicseq2-norm takes raw read count files (typically in BED or Wiggle format) and applies GC-content and mappability corrections using a predefined bias model to produce normalized coverage values.
  
- **Bias Model Input**: The tool requires a pre-computed bias model file (generated by bicseq2-build or similar) containing per-window GC percentages and mappability scores that the normalization algorithm uses to adjust read counts.

- **Window-Based Processing**: Normalization operates on fixed-size genomic windows (default 1kb, configurable via --window-size), processing each window independently and outputting corrected counts in the same format as input.

- **Output Format**: The normalized output is written in Wiggle (Wig) format by default, with chromosome, start, end, and normalized score columns, suitable for direct use in downstream CNV segmentation tools.

## Pitfalls

- **Mismatched Genome Build**: Using a bias model built for a different genome build (e.g., hg19 vs hg38) will produce incorrect normalized values and potentially mask real CNVs or create false positive signals.
  
- **Window Size Mismatch**: If the normalization window size differs from the window size used in the original bias model generation, the correction factors will be applied incorrectly, leading to over- or under-normalized regions.

- **Insufficient Memory for Large Genomes**: Attempting to normalize whole-genome data without sufficient RAM can cause the tool to fail silently or truncate output, resulting in incomplete coverage maps that will affect downstream CNV calling accuracy.

- **Input Format Errors**: Providing read counts in incompatible formats (e.g., BEDGraph instead of expected BED or Wiggle) will cause parsing failures. The tool expects zero-based start coordinates in BED files, but one-based in Wiggle files.

## Examples

### Normalize a BAM-derived read count file using a pre-built bias model
**Args:** --input sample.reads.wig --bias-model hg19.bias --output sample.norm.wig
**Explanation:** This applies GC and mappability corrections from the hg19 bias model to the input Wiggle file, producing a normalized output file suitable for CNV segmentation.

### Adjust normalization window size to 500bp for high-resolution analysis
**Args:** --input sample.reads.wig --bias-model hg19.bias --window-size 500 --output sample.norm.wig
**Explanation:** Setting a smaller window size increases resolution but requires a correspondingly built bias model; mismatched sizes lead to incorrect normalization.

### Specify a specific chromosome for targeted normalization
**Args:** --input sample.reads.wig --bias-model hg19.bias --chr chr22 --output sample.chr22.norm.wig
**Explanation:** Specifying a single chromosome reduces memory usage and runtime when working on particular genomic regions of interest.

### Enable verbose logging to debug normalization issues
**Args:** --input sample.reads.wig --bias-model hg19.bias --output sample.norm.wig --verbose
**Explanation:** Verbose mode outputs detailed progress information, helpful for identifying where normalization fails or produces unexpected values.

### Use median normalization instead of default mean-based approach
**Args:** --input sample.reads.wig --bias-model hg19.bias --norm-method median --output sample.norm.wig
**Explanation:** Median normalization is more robust to outliers and extreme copy number events, providing more stable baseline estimates for CNV detection.

### Skip bias correction and output raw counts in different format
**Args:** --input sample.reads.bed --bias-model hg19.bias --no-correct --output-format bedgraph --output sample.raw.bg
**Explanation:** Disabling correction allows comparison of raw vs normalized counts or enables use of custom downstream correction methods.
