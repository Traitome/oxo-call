## Concepts

- **Graph Input Format**: bubblefinder processes assembly graphs in GFA (Graphical Fragment Assembly) format or custom binary formats. The input graph must contain node sequences, edge connections, and bubble annotated regions. Graphs from SOAPdenovo, ABySS, or Velvet can be converted to GFA using auxiliary tools before analysis.

- **Bubble Identification Algorithm**: The tool traverses the graph to locate bubble structures—two or more node-disjoint paths connecting the same start and end nodes. Each bubble represents alternative paths that diverge at a specific graph position and reconverge downstream. The algorithm uses path coverage, k-mer multiplicity, and node frequency thresholds to distinguish true biological variants from sequencing errors.

- **Output Reporting**: Results are written in a tab-delimited format with columns for bubble ID, start node, end node, alternative path details, coverage metrics, and predicted variant type (heterozygous SNP, insertion, deletion, or complex). An optional VCF output can be generated for direct integration with variant calling pipelines.

- **Filtering Parameters**: bubblefinder supports read depth and coverage ratio filters to reduce false positives. The minimum coverage threshold (default: 3) excludes low-frequency paths that likely represent sequencing noise. The bubble size limit prevents detection of excessively large structures that indicate assembly collapse rather than simple variants.

## Pitfalls

- **Insufficient Graph Coverage**: Providing a graph built from low-coverage sequencing data produces spurious bubbles that represent incomplete assembly rather than biological variants. Bubbles detected in graphs with mean coverage below 10x should be validated against original read alignments before interpretation.

- **Wrong Graph Format**: Attempting to load non-GFA graphs without proper conversion causes parsing failures or silent errors. Always verify the input graph format using the `--check-format` flag or auxiliary file inspection tools before running bubblefinder.

- **Aggressive Filtering**: Setting coverage thresholds too high (e.g., `--min-coverage 20`) eliminates true heterozygous bubbles that have allele-imbalanced coverage in haploid regions. The optimal threshold depends on the sequencing depth and expected heterozygosity level—start with defaults before adjusting.

- **Ignoring Bubble Size Distribution**: Large bubbles (>500 bp) often indicate repeat-induced graph collapse rather than simple variants. Treating all detected bubbles equally without size filtering leads to false positive variant calls in repetitive genomic regions.

## Examples

### Detect bubbles in a standard assembly graph
**Args:** -g assembly_graph.gfa -o bubbles_output.tsv
**Explanation:** Reads the GFA assembly graph and writes all detected bubble structures to the output file for initial exploration without filtering.

### Find bubbles with minimum coverage threshold
**Args:** -g assembly_graph.gfa --min-coverage 5 -o high_conf_bubbles.tsv
**Explanation:** Applies a coverage filter to exclude low-frequency paths likely caused by sequencing errors, outputting only bubbles with coverage at least 5 across alternative paths.

### Limit bubble size to detect simple variants
**Args:** -g assembly_graph.gfa --max-bubble-size 200 -o small_bubbles.tsv
**Explanation:** Restricts detection to small bubble structures under 200 bp, focusing on simple SNPs and small indels rather than complex assembly artifacts.

### Export results in VCF format
**Args:** -g assembly_graph.gfa -o bubbles.vcf --format vcf
**Explanation:** Generates VCF output compatible with downstream variant analysis pipelines, enabling direct integration with tools like GATK or bcftools.

### Adjust path divergence ratio
**Args:** -g assembly_graph.gfa --divergence-ratio 0.3 -o divergent_bubbles.tsv
**Explanation:** Filters bubbles where alternative path coverage differs by more than 30%, focusing on balanced heterozygous variants rather than error-dominated paths.

### Process a compressed graph file
**Args:** -g assembly_graph.gfa.gz -o bubbles.tsv
**Explanation:** Automatically detects and decompresses gzip-compressed input, reducing storage requirements for large assembly graphs.

### Run with verbose logging for debugging
**Args:** -g assembly_graph.gfa -o bubbles.tsv --verbose
**Explanation:** Enables detailed logging of graph traversal and bubble detection steps, useful for troubleshooting unexpected output or performance issues.
