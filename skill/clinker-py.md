## Concepts

- **Input formats**: Clinker accepts gene cluster files in multiple formats including antiSMASH JSON output (`*.json`), GenBank flat text (`.gbk` or `.gb`), and Sequeton tabular format (`.tbl`). All input files should contain annotated coding sequences with product type annotations for meaningful comparisons.

- **Similarity scoring**: The tool calculates multiple similarity metrics between gene clusters including Jaccard index (gene presence/absence), Smith-Waterman local alignment scores for amino acid sequences, and Bit Score comparisons using DIAMOND or BLAST. These scores are output as JSON statistics and can be used to filter orthologous gene pairs.

- **Output files**: Clinker generates alignment triples (`*.txt`) showing which genes in cluster A correspond to genes in cluster B with alignment scores, a combined results JSON containing all pairwise comparisons, and optional filter files (`*.json.filter`) for downstream visualization or filtering in clinker-plot.

- **BLAST/DIAMOND backend**: For amino acid alignments, clinker can use either BLAST+ (via `makeblastdb` and `blastp`) or DIAMOND as the sequence comparison engine. DIAMOND is significantly faster for large-scale comparisons but must be installed separately and referenced via the `--diamond` flag.

- **Filter threshold**: The `--filter` argument controls the minimum alignment score (normalized percentage) for including a gene pair in the output. Setting this too low (e.g., 25) produces noisy results with weak matches; setting it too high (e.g., 90) may exclude genuinely conserved genes with moderate sequence divergence.

## Pitfalls

- **Missing sequence data in JSON input**: If using antiSMASH JSON files, ensure they contain the full nucleotide sequences (`"sequences"` section). Clinker may crash or produce empty alignments if gene sequences are absent from the JSON, especially when JSON files are truncated or generated by older antiSMASH versions.

- **Inconsistent gene identifiers across files**: When comparing clusters from different sources, genes with identical names but different sequences (or vice versa) can cause confusion in the alignment output. Clinker uses sequence-based matching primarily, so duplicate gene names are accepted but may confuse downstream manual interpretation.

- **Omitting required output directory**: The output directory (`-o`) is mandatory for saving results. If omitted, clinker writes alignment triples to stdout but the combined JSON results file is not saved, requiring you to re-run the comparison to regenerate the full comparison data.

- **Incompatible BLAST version for amino acid mode**: When running in amino acid comparison mode (`-a aa`), BLAST+ must be in the system PATH. If BLAST+ is not installed or the database step (`makeblastdb`) fails, clinker aborts with a non-obvious error about missing executables.

- **Large cluster comparisons with high memory usage**: Comparing many large gene clusters simultaneously (e.g., >50 clusters with >50 genes each) can consume substantial RAM. On memory-constrained systems, process clusters in batches rather than providing all files at once to avoid being killed by the operating system.

## Examples

### Compare two antiSMASH JSON gene cluster files
**Args:** `-o results_dir cluster1.json cluster2.json`
**Explanation:** Compares two gene clusters from antiSMASH JSON output files, saves all pairwise gene alignments and similarity scores to the specified output directory.

### Compare multiple GenBank files with nucleotide alignment
**Args:** `-o results_dir -a nucl cluster1.gbk cluster2.gbk cluster3.gbk`
**Explanation:** Aligns gene sequences using nucleotide (nucl) mode across three GenBank files, computing pairwise similarities between all cluster pairs and saving results to the output directory.

### Compare clusters with high stringency filter
**Args:** `-o results_dir --filter 75 cluster1.json cluster2.json`
**Explanation:** Runs the comparison but only includes gene pairs with alignment scores of 75% or higher in the output, reducing noise from weakly conserved hypothetical genes.

### Compare clusters with DIAMOND acceleration
**Args:** `-o results_dir -a aa --diamond diamond_binary --db-dir ./diamond_dbs cluster1.json cluster2.json`
**Explanation:** Performs amino acid alignment using DIAMOND instead of BLAST+ for faster processing on large datasets, with the specified DIAMOND binary and pre-built database directory.

### Compare clusters outputting triples only to stdout
**Args:** `-a nucl cluster1.json cluster2.json`
**Explanation:** Runs a quick comparison without specifying an output directory, printing alignment triples to standard output for immediate inspection of which genes map between clusters.

### Compare clusters with detailed JSON results
**Args:** `-o results_dir -c --json --filter 50 cluster*.json`
**Explanation:** Processes multiple JSON files using glob expansion, saves a detailed combined JSON file (`results.json`) containing all comparison metrics, and applies a moderate 50% score filter to include more divergent but genuine homologs.
