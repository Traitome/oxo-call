## Concepts

- Booster maps long reads (ONT FASTA/FASTQ) to a reference sequence using seed-based heuristics followed by banded dynamic programming extension, which is computationally efficient for reads exceeding 1 kb.
- Input reads must be in FASTA or FASTQ format with quality scores (PHRED+33 encoding); reference sequences are accepted as plain text or indexed FASTA with associated .fai index files generated by a companion indexer.
- Output is written as aligned reads in MAF (Multiple Alignment Format) or SAM-like format depending on the specified output flag; the alignment includes CIGAR strings describing insertions, deletions, and substitutions relative to the reference.
- Booster builds an indexed reference structure automatically on first run using an embedded indexer (similar to bowtie2-build), which is stored as multiple auxiliary files adjacent to the reference FASTA.
- The tool supports threading via `-t N` to parallelize independent read alignments across multiple cores; memory consumption scales roughly linearly with both read length and reference genome size.

## Pitfalls

- Specifying an unindexed reference FASTA without a pre-existing .fai index causes Booster to build the index on the fly, which can take several minutes for mammalian-sized genomes and will fail if write permissions are absent in the reference directory.
- Using FASTQ input without PHRED quality scores (legacy Solexa format) produces silently incorrect alignment scores because Booster assumes linear PHRED scaling, leading to erroneous base quality interpretations in output.
- Setting `-t` to a value exceeding the available physical CPU cores causes thread oversubscription and severe performance degradation on most HPC schedulers; always verify core availability before increasing thread count.
- Booster assumes uniform per-base error rates for ONT reads and does not model context-dependent errors; homopolymer runs in the reference may accumulate alignment artifacts that require post-filtering.
- Omitting the `-o` output flag directs all results to stdout, which can truncate large alignment outputs on terminal buffers and lose trailing alignment records when redirected via pipes.

## Examples

### Map a set of ONT FASTQ reads to a bacterial reference genome

**Args:** `-r ecoli_ref.fasta -i reads.fastq -o alignments.maf`
**Explanation:** Maps each read in reads.fastq to the indexed E. coli reference, writing resulting alignments in MAF format to the specified output file.

### Boost ONT reads with automatic reference indexing and 8 threads

**Args:** `-r hg38_ref.fasta -i ont_reads.fq -o results.sam -t 8`
**Explanation:** Performs read boosting against the human reference with multithreaded execution, creating the index automatically if absent and outputting SAM-format alignments.

### Align long reads with elevated gap penalty to suppress indel artifacts

**Args:** `-r assembly.fasta -i raw_reads.fastq -o strict_alignments.maf -g 12`
**Explanation:** Uses a gap open penalty of 12 to penalize insertions and deletions more heavily, producing alignments that favor substitutions over indels for bias toward assemblies.

### Process a small number of reads for debugging alignment output

**Args:** `-r test_ref.fa -i debug_reads.fastq -o debug_out.maf -v`
**Explanation:** Runs in verbose mode to print detailed per-read alignment statistics to stderr, useful for inspecting why a specific read fails to map.

### Generate boosted alignments with soft clipping enabled at read ends

**Args:** `-r ref_genome.fasta -i nanopore_reads.fastq -o soft_clipped.sam --soft-clip`
**Explanation:** Enables soft clipping of terminal alignment portions rather than hard clipping, preserving the original read sequences in the SAM output for downstream tools like variant callers.
