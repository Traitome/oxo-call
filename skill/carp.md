## Concepts

- **Input format**: CARP accepts unaligned BAM files (`.bam`) containing SMRT reads, typically generated by PacBio RSII or Sequel instruments. The reads must be in coordinate-sorted or queryname-sorted order with valid CIGAR strings and MD tags for variant detection.
- **Phased consensus assembly**: CARP constructs separate consensus sequences for each detected haplotype (phased variant clusters), making it ideal for distinguishing co-infecting viral strains or low-frequency mutations below standard variant callers' detection thresholds.
- **Output artifacts**: The tool produces FASTA consensus sequences (one per haplotype), CSV variant call files with allele frequencies and coverage statistics, and a summary JSON report containing run metrics, consensus quality scores, and phased read counts.
- **Reference requirement**: A reference genome must be provided as a plain FASTA file (`.fa` or `.fasta`). The reference should be closely related to the input reads (typically same species or strain) as CARP aligns reads directly to the reference without de novo assembly.
- **Filtering by coverage and frequency**: Reads and variants are filtered based on configurable minimum coverage (default: 5 reads per strand) and minimum allele frequency (default: 0.01) thresholds, which must be tuned for your experiment's sequencing depth and expected variant frequencies.

## Pitfalls

- **Mismatched reference strain**: Providing a reference from a distantly related strain (e.g., wrong viral subtype or species) causes reads to fail alignment, resulting in no consensus calls and empty output files. Always verify reference provenance matches your sample.
- **Insufficient sequencing depth**: With shallow SMRT coverage (fewer than 20 reads per site), CARP produces consensus sequences with high uncertainty, and low-frequency variants may be missed entirely or called with spurious alleles. Increase mean coverage above 50x for reliable quasispecies profiling.
- **Misconfigured frequency threshold**: Setting `--min-freq` too high (e.g., 0.1) excludes legitimate low-frequency variants present in minor quasispecies populations, defeating the purpose of using CARP for sensitive detection. Use thresholds matching your biological question.
- **Ignoring strand bias in variants**: CARP flags strand-biased variants, but proceeding with biologically implausible strand-skewed calls (e.g., 100% forward reads supporting an allele) can produce false positives from systematic sequencing artifacts. Always review strand metrics in the CSV output.

## Examples

### Generate consensus sequences with default parameters
**Args:** -i sample.unmapped.bam -r viral_ref.fa -o carp_results
**Explanation:** Runs CARP with default filtering thresholds to produce consensus sequences and variant calls for all detected haplotypes, saving outputs to the specified directory.

### Increase sensitivity for rare variant detection
**Args:** --min-freq 0.005 --min-cov 3 --min-reads 2 -i sample.bam -r ref.fa -o sensitive_output
**Explanation:** Lowers frequency and coverage thresholds to detect ultra-rare variants (0.5% allele frequency) present in minor quasispecies populations, at the cost of increased false-positive risk.

### Limit analysis to a genomic region of interest
**Args:** --region "NC_012960.1:1000-5000" -i sample.bam -r ref.fa -o roi_results
**Explanation:** Restricts analysis to a specific chromosomal interval, reducing runtime and avoiding calls from off-target read mappings while focusing on the biologically relevant locus.

### Output phased haplotypes with read assignments
**Args:** --phased --phase-cov 10 -i sample.bam -r ref.fa -o phased_results
**Explanation:** Generates separate consensus sequences per haplotype and exports per-haplotype coverage statistics, enabling reconstruction of which reads support each detected strain variant.

### Adjust minimum read length filter
**Args:** --min-read-length 500 -i sample.bam -r ref.fa -o long_reads_only
**Explanation:** Filters input reads to exclude short subread fragments that may cause spurious alignments, improving consensus quality when working with fragmented SMRT data or shorter insert sizes.

### Generate detailed variant report for downstream validation
**Args:** --report-variants --all-variants -i sample.bam -r ref.fa -o variant_report
**Explanation:** Exports comprehensive CSV and JSON reports including all called variants withstrand bias metrics, allele frequencies, read support counts, and quality scores for manual review and downstream validation.

### Control number of parallel threads
**Args:** -t 16 -i sample.bam -r ref.fa -o parallel_results
**Explanation:** Allocates 16 threads to speed up alignment and variant calling steps on multi-core systems, reducing wall-clock time for large BAM files with high read counts.
