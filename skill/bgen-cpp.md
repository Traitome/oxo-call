## Concepts

- **BGEN binary structure**: BGEN stores genotype probabilities as fixed-length or variable-length blocks per variant. Each variant entry contains: rsID, chromosome, position, reference/alternate alleles, and probability arrays. Unphased genotypes use 2 probabilities per sample, while phased data uses sequential allele probabilities. The toolkit reads this binary layout directly without decompression to external buffers.

- **Index files (.bgi)**: Companion index files (.bgi) enable random-access queries on large BGEN files. These index files are automatically generated by `bgen-cpp-idxgen` and store offset pointers for each variant block. Without an index file, sequential scanning is required for all queries, which is impractical for cohort-scale files containing millions of variants.

- **Sample files (.sample)**: Many BGEN operations require a corresponding `.sample` text file listing participant identifiers and optional covariates. The format mirrors Oxford `.sample` files with tab or space-delimited columns. Not all BGEN files carry embedded sample identifiers; external `.sample` files must be specified separately via the `--sample` flag for dosage and allele frequency calculations.

- **Output formats**: Results can be directed to stdout for piping, or written to files. For variant lists, output defaults to tab-separated values (TSV) with a header row. Statistical summaries may be emitted in JSON when the `--json` flag is applied. Understanding output format assumptions prevents downstream parsing failures in automated pipelines.

## Pitfalls

- **Forgetting the index file**: Running queries like `--range` or `--from-id` on a BGEN file without a pre-existing `.bgi` index causes the tool to either abort with a read error or silently perform a slow sequential scan on the entire file. Always generate the index file upfront using `bgen-cpp-idxgen` for files exceeding a few thousand variants.

- **Mismatched sample file**: Specifying a `.sample` file that does not correspond to the BGEN file (different sample count or ordering) produces nonsensical dosage and frequency values. The tool does not validate sample-to-genotype alignment, so garbage-in-garbage-out behavior occurs silently. Cross-check sample counts before running statistical commands.

- **Confusing phased vs. unphased flags**: The `--phased` flag controls whether output includes phased haplotype strings or dosage values. Using `--phased` on an unphased BGEN file produces all-zero output without warning, because no phased data exists in the source. Verify the phasing status of your BGEN file via `bgen-cpp-info` before selecting the output mode.

- **Loss of precision with default integer conversion**: By default, probability values are rounded to 4 decimal places in TSV output. For downstream pipelines requiring higher precision (e.g., post-GWAS meta-analysis with fine-mapping), use the `--precision 8` flag to avoid rounding-induced bias. This oversight is particularly damaging for rare variant analyses.

- **Inconsistent chromosome naming**: BGEN files may store chromosomes as integers (1, 2, ...) or strings ("chr1", "chr1_autosome"). Queries using `--chromosome` with a mismatched naming convention return empty results with exit code 0, which pipeline scripts may interpret as a valid empty result set rather than a lookup failure.

## Examples

### List all variants in a BGEN file
**Args:** `info data/example.bgen`
**Explanation:** The `info` subcommand reads the BGEN header and prints variant metadata (count, flags, compression type), confirming file integrity before downstream operations.

### Extract variants within a genomic range
**Args:** `query data/cohort.bgen --chromosome 22 --from 30000000 --to 35000000 --output-range variants_22q.bgen`
**Explanation:** The `query` subcommand with range filters writes a new BGEN containing only variants overlapping the specified genomic interval, enabling locus-specific analyses without loading the full dataset.

### Compute allele frequencies for all variants
**Args:** `afreq data/cohort.bgen --sample data/samples.txt --out frequencies.tsv`
**Explanation:** The `afreq` command computes alternate allele frequencies per variant using sample-level probability sums, outputting a tab-delimited table used directly in QC filtering or lambda inflation checks.

### Export dosages to a VCF-style text file
**Args:** `export data/cohort.bgen --sample data/samples.txt --format dosage --precision 8 --out dosages.tsv`
**Explanation:** The `export` command with dosage format extracts probability-weighted genotype calls at full precision for integration with R or Python statistical environments requiring numeric genotype matrices.

### Identify and export samples missing genotype data
**Args:** `missingness data/cohort.bgen --sample data/samples.txt --threshold 0.1 --out missing_samples.tsv`
**Explanation:** The `missingness` command flags samples whose proportion of missing ( homozygous reference-equivalent ) probabilities exceed the supplied threshold, facilitating sample-level QC prior to imputation or association testing.

### Generate a BGEN index file for random access
**Args:** `idxgen data/cohort.bgen`
**Explanation:** The `idxgen` command creates a `.bgi` companion index enabling O(1) offset lookups, which is prerequisite for efficient `--from-id` and `--range` queries on large-scale GWAS files.
